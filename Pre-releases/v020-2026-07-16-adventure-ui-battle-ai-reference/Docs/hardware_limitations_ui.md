# HMM2 VDAC2 — аппаратные ограничения UI (отклонения от OpenHMM2 по железу)

Список ведётся автономно. Сюда — только то, что НЕ удаётся сделать «как в оригинале»
из-за железа TS-Conf Z80 14 МГц + FT812/VDAC2, с причиной. Всё остальное — строго по OpenHMM2.

Бюджеты FT812: RAM_DL 8192 байт/кадр, CMD-FIFO 4096 байт/кадр, RAM_G ~1 МБ.

Формат: `- [ЭЛЕМЕНТ] — отклонение — причина (бюджет/возможность железа).`

## Подтверждённые ограничения (из аудита)
- **[radar] фиксированный масштаб 4px/тайл** (144÷36) — нет динамического зума 1:1…4:1 как в OpenHMM2.
  Причина: карта зафиксирована 36×36; MIN(144px радара)/MAX(тайлов) не масштабируется без потери. Для карт
  ≠36×36 потребовалась бы переработка MinimapTileColorTable и MINIMAP_TILE_PX.
- **[icons] маркер выбора (рамка 56×32 cyan) per-frame дорог** как LINE_STRIP (~30 Б/кадр CMD-FIFO).
  Дешевле — запечь маркер в композит PORTXTRA+border (1 BITMAP_SOURCE+VERTEX2F). Решение: запекать.
- **[buttons] нет затемнения disabled/inactive кнопок** (оригинал RGB 96,96,96). per-button COLOR_RGB снят
  ради экономии 8×4Б=32Б/кадр CMD-FIFO (зарезервировано под курсор; курсор иначе вылетает за бюджет 4096).
  UI_BtnColors (render.asm:1722) определён, но не используется. Альтернатива: предзатемнённые кадры в ассете.
- **[cpanel/buttons] нет per-sprite AlphaBlit(128)** — глобальный COLOR_RGB(255,255,255) один раз.
  Причина: per-sprite alpha = доп. команды/кадр CMD-FIFO. Кнопки рисуются непрозрачно (PALETTED4444).

## Заметки
- Бюджеты RAM_DL/CMD-FIFO — общий потолок кадра. Большинство «missing»-фич укладываются (список икон ~96 Б/кадр),
  но СУММА маркеров+затемнений+скроллбара требует учёта; при нехватке — запекать статику в ассеты, а не рисовать per-frame.
- НЕ ограничение: масштаб мини-карты 4px/тайл совпадает с оригиналом для 36×36; «чёрный радар» на старте = туман (норма).
