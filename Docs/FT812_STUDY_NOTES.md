# Краткая выжимка по FT812 / VDAC2 для HMM2

Источник: `Docs\uchebnik_tsconf_vdac2.md`.

## Цель проекта

HMM2 делается под VDAC2 / FT812, целевой режим `640x480`.

Не используем ZX border, ZX bitmap или TS-Conf fallback как видимую диагностику,
если это не будет отдельно запрошено.

## Старт FT812

Порядок старта:

1. Проверить VDAC2 через `STATUS & 7 == 7`.
2. Выполнить `FT_BOOT_UP`.
3. Выполнить `FT_CMD_RESET`.
4. Задать режим `FT_RESOLUTION VM_640_480_74Hz, ResolutionWidthPtr`.
5. Записать пустой display list через `FT.WriteDL`.
6. Сделать `DLSWAP`.
7. Включить `Video_Setting VID_FT812 | VID_NOGFX`.

## Важная правка для HMM2

HMM2 стартует с `DI`, поэтому задержка через `HALT` внутри `FT_BOOT_UP`
останавливает CPU. В локальной копии `Docs\TSLib\Include\FT\812 Macro.inc`
`FT_DELAY` заменен на busy-loop.

## Display List

RAM_DL маленькая, около 8 КБ, поэтому кадр должен быть компактным.

Display List надо строить как команды FT812, затем завершать `DISPLAY()` и
делать `DLSWAP`. Состояние bitmap-параметров у FT812 сохраняется, поэтому
надо явно задавать `BITMAP_HANDLE`, `BITMAP_SOURCE`, `BITMAP_LAYOUT`,
`BITMAP_SIZE` при смене набора тайлов/атласа.

## Архитектурное следствие

Core держит только минимальный старт, FT812 init и диспетчер.

Декодеры, загрузчики AGG/TIL/ICN, подготовка атласов, рендер режимов карты,
города и боя должны жить в страницах/оверлеях/пакетах.

## Следующий практический шаг

Для экрана карты сначала сделать статический FT812 viewport:

- взять `SKIRMISH.map.bin`;
- извлечь и разобрать `GROUND32.TIL`;
- подготовить маленький тестовый атлас в RAM_G;
- вывести видимую область карты через display list.
